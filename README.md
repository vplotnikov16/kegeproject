# НАЗВАНИЕ_ИС

Основной скрипт для запуска проекта - `run.py`. Для запуска без ошибок нужно убедиться, что установлены все зависимости.
```bash
python run.py
```

Для удобного запуска и автоматической настройки окружения есть скрипт `run_in_venv.py`. 
```bash
python run_in_venv.py
```
Работает через poetry. Желательно запускать его через системный интерпретатор.
Если хочется запустить из интерпретатора уже установленного окружения проекта, то нужно убедиться, что системный интерпретатор лежит в PATH и доступен (нужно для управления poetry).
В теле функции main есть флаг `is_dev_env`, который регулирует, нужно ли ставить dev-зависимости.
